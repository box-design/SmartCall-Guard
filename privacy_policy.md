# 智能来电守护（SmartCall Guard）隐私政策

**最后更新日期：2026年5月5日**

## 概述

智能来电守护（以下简称"本应用"）高度重视用户隐私保护。本隐私政策旨在向您说明本应用如何收集、使用和存储您的个人信息。

## 信息收集

### 通讯录信息
- **用途**：仅用于将通讯录号码导入本地白名单，避免误拦截熟人电话。
- **存储**：通讯录号码经标准化处理后存储于设备本地数据库。
- **上传**：本应用 **不会** 将通讯录信息上传至任何服务器。

### 位置信息
- **用途**：仅用于本地归属地匹配，判断来电号码归属地与您当前位置的关系（如同城、同省等），实现归属地智能拦截。
- **存储**：位置信息缓存于设备本地内存（默认 30 分钟有效期），不上传服务器。
- **精度**：使用网络定位和 GPS 定位，精度取决于设备设置。
- **上传**：本应用 **不会** 将位置信息上传至任何服务器。

### 通话记录
- **用途**：记录被拦截的来电号码及相关信息（归属地、拦截原因），方便您查看拦截历史及误拦截恢复。
- **存储**：拦截记录存储于设备本地数据库。
- **上传**：本应用 **不会** 将拦截记录上传至任何服务器。

### 来电号码
- **用途**：系统级来电筛选服务会在来电时实时检查来电号码，用于规则匹配判断。
- **存储**：仅被拦截的来电号码会记录到本地日志；正常来电号码不会被记录。
- **上传**：本应用 **不会** 将任何来电号码上传至任何服务器。

## 数据共享

本应用 **不会** 与任何第三方共享您的任何数据。所有数据处理均在设备本地完成。

## 可选云端增强功能

本应用提供可选的云端增强功能（默认关闭），包括：
- 号段数据库增量更新
- 云端号码标记库查询

启用此类功能时，仅通话号码（匿名化）会发送至服务器进行标记查询。您可随时在设置中关闭此功能。

## 数据安全

- 所有本地数据存储于应用私有目录，其他应用无法访问。
- 本应用不会在后台持续收集位置信息，仅在来电决策需要时使用缓存位置。

## 权限说明

本应用申请的各项权限及用途：

| 权限 | 用途 |
|------|------|
| CALL_SCREENING | 核心来电拦截功能 |
| READ_PHONE_STATE | 读取来电状态 |
| READ_CONTACTS | 导入通讯录到白名单（用户主动操作） |
| ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION | 归属地匹配 |
| ACCESS_BACKGROUND_LOCATION | 后台来电时归属地匹配 |
| FOREGROUND_SERVICE | 保活服务，确保来电拦截持续运行 |
| POST_NOTIFICATIONS | 前台服务通知 |
| RECEIVE_BOOT_COMPLETED | 开机自启动服务 |
| REQUEST_IGNORE_BATTERY_OPTIMIZATIONS | 防止系统杀死服务 |

## 用户权利

- 您可随时在设置中导出或清空所有拦截记录
- 您可随时在设置中关闭归属地拦截以停止位置信息使用
- 您可随时删除已导入的白名单号码
